# ASCII Filter

```
Usage: asciifilter [<options>] <file>

  Convert images to ASCII art using any set of characters you choose.

   • Can automatically find out the brightness of characters.
   • Can adapt to character sets with different aspect ratios. (For example
     Korean or Hebrew have different aspect ratio compared to Latin).
   • Parallerized using Kotlin coroutines for faster execution on large images.

  ▎ Use white or transparent background for best results.

  Examples:

Options:
  -a, --alphabet=<text>  Set of characters (in arbitrary order) that should be
                         used to generate target ASCII image. Any UTF-8
                         characters can be used, as long as they all have same
                         aspect ratio. Default:
                         $@B%8&WM#*oahkbdpqwmZO0QLCJUYXzcvunxrjft/\|()1{}[]?-_+~<>i!lI;:,"^`'.
  -w, --width=<int>      The number of character width of the result image.
                         Default: 50
  -f, --font=<text>      Name of the font to use when figuring out character
                         brightness and aspect ratio. Monospace fonts should be
                         used to get correct results. Default: Monospaced
  -v, --verbose          Enable debug logging.
  --fontSize=<int>       The font size that will be used to calculate how
                         bright or dark each character is. Lower values will
                         lead to faster startup and lower memory usage.
                         However, lowering this too much will result in worse
                         ASCII image quality. Default: 10
  -h, --help             Show this message and exit

Arguments:
  <file>  Image to convert to ASCII art. Use white background for best results.
          Supported formats are: jpg, png, gif, bmp, wbmp
```
